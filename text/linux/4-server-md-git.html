git的提交commit指令

commit指令有很多种用法

git commit -m "提交的描述" 例如：”added homepage”

touch newfile #新建文件
git status    #查看git状态

i     #vim的插入模式

:wq #vim的保存退出
git status  #已经没有了，git直接帮你提交了

如果有不想提交的文件

例如

vim todo.md #新建todo.md这个文件
add some pictures #编辑内容
:wq #保存退出
vim .gitignore #新建文件
todo.md #编辑内容：需要让git忽略的文件名
:wq #保存退出
git status #查看状态，只有一个.gitignore文件，todo.md并没有追踪
git add .  #添加这个文件，搞定
git commit -m "added gitignore" #提交并告诉git这个文件的描述

git一开始追踪一个文件，后面再添加到忽略文件里面也不管用，除非告诉git不用再追踪了

告诉git不用再追踪一个文件

git rm --cached newfile #告诉git不用再帮我追踪newfile这个文件了

git add .  #添加一次
git commit -m "removed newfile"  #提交一次，并告诉git描述

git分支

git branch pictures #添加一个叫照片的分支
git checkout pictures #切换到照片这个分支

这两个分支之间切换的效果，可以再图形界面看到变化，再切换的瞬间，文件全部都变到了另一个分支的状态

合并分支

git merge pictures #把pictures这个分支合并到master这个主分支下
git 合并时候的冲突问题，可以自己上网查一下

查看git有几个分支

git branch

删除分支

即使合并了的分支，git还是会给你留着的，如果想要删除怎么办?
git branch -d pictures #删除pictures这个分支
如果你还有些更改没有合并，如果删除文件会丢失，git会不让删除
git branch -D pictures #强制git删除pictures这个分支，不管会不会丢失文件

把更改提交到github

git remote add origin https://github.com/....  #告诉git这个项目在往上的地址在哪
git push --set-upstream origin master #提交到网上，master就是主分支
--在这里git会提示你输入用户名密码，输入完就会提交到github上了--
--如果想要git记住用户名密码--
git config credential.helper store
--这句话写完后在输入git push 输入用户名密码后，下一次就不用在输入了--
